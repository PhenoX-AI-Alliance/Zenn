近年、IoTデバイスとサーバーレスアーキテクチャを組み合わせた環境モニタリングシステムが注目を集めています。従来のクラウドサービスでは、データベースやサーバーの維持費が課題となりがちでしたが、Cloudflareの提供するエッジコンピューティングを活用することで、驚くほど低コストかつ堅牢なモニタリング基盤を構築することが可能です。

本記事では、Cloudflare WorkersとKV（Key-Valueストア）のみを使用して、センサーデータを収集・可視化する仕組みを解説します。

---

## システムアーキテクチャ

1.  **データ収集**: ESP32等のデバイスからCloudflare WorkersのエンドポイントへPOSTリクエストを送信。
2.  **データ保存**: Workersが受け取ったデータをCloudflare KVに保存。
3.  **データ閲覧**: Cloudflare Pages（またはWorkers）で生成したHTML/JSからKVを読み込み、グラフ化して表示。

---

## 実装例

### 1. データの保存 (Cloudflare Workers)

センサーからのデータを受け取り、KVに保存するコードです。

```javascript
// wrangler.tomlでKV_NAMESPACEをバインドしておく必要があります
export default {
  async fetch(request, env) {
    if (request.method === "POST") {
      const data = await request.json();
      const timestamp = new Date().toISOString();
      // タイムスタンプをキーにして保存
      await env.SENSOR_DATA.put(timestamp, JSON.stringify(data));
      return new Response("OK", { status: 200 });
    }
    return new Response("Method not allowed", { status: 405 });
  }
};
```

### 2. データの可視化 (HTML/JS)

フロントエンド側でKVからデータを取得し、Chart.jsなどで描画します。

```html
<!-- index.html -->
<canvas id="myChart"></canvas>
<script>
  async function fetchData() {
    // KVからデータを取得するWorkersエンドポイントを叩く
    const response = await fetch('/api/data');
    const data = await response.json();
    
    // Chart.jsによる描画ロジック
    const ctx = document.getElementById('myChart').getContext('2d');
    new Chart(ctx, {
      type: 'line',
      data: {
        labels: data.map(d => d.time),
        datasets: [{ label: 'Temperature', data: data.map(d => d.temp) }]
      }
    });
  }
  fetchData();
</script>
```

## なぜCloudflareなのか

*   **圧倒的なコストパフォーマンス**: Cloudflare Workersの無料枠は非常に寛大であり、小規模な環境モニタリングであれば無料で運用可能です。
*   **メンテナンスフリー**: サーバーのパッチ適用やスケーリングを意識する必要がありません。
*   **低遅延**: エッジで動作するため、地理的に分散したセンサーからのデータ収集も高速です。
