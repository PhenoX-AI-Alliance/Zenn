## 1. なぜ「環境負荷低減」にDDDが必要なのか

環境負荷を低減するWebアプリとは、単にサーバーを軽くするだけではありません。**「地球環境データ」という複雑なドメイン**を正確にモデリングし、計算資源の無駄を省きつつ、持続可能な意思決定を支援するロジックを構築する必要があります。

DDDを採用することで、ビジネスロジックと環境負荷の計算ロジックを分離し、保守性と最適化の両立が可能になります。

## 2. 実装解説：`src/domain/resilience_model.ts`

環境レジリエンス（回復力）を計算するためのドメインモデルを紹介します。このモデルは、環境データの変動に対してシステムがどのように適応するかを定義します。

```typescript
// src/domain/resilience_model.ts

export interface EnvironmentalMetrics {
  carbonFootprint: number; // kgCO2e
  energyEfficiency: number; // 0.0 - 1.0
}

export class ResilienceModel {
  constructor(
    private readonly metrics: EnvironmentalMetrics,
    private readonly threshold: number = 0.8
  ) {}

  /**
   * 環境負荷を最小化するための最適化ロジック
   * ドメインの不変条件として、効率が閾値を下回る場合はアラートを発行
   */
  public calculateOptimizationImpact(): number {
    const efficiency = this.metrics.energyEfficiency;
    
    if (efficiency < this.threshold) {
      // 効率改善のための計算リソース再配分ロジック
      return (this.threshold - efficiency) * this.metrics.carbonFootprint;
    }
    
    return 0;
  }
}
```

### 実装のポイント
*   **カプセル化**: `EnvironmentalMetrics`を値オブジェクトとして扱い、不変性を保っています。
*   **ドメインロジックの分離**: `calculateOptimizationImpact`メソッドは、外部APIやDBに依存せず、純粋な計算ロジックとして定義されています。これにより、Claude Codeを用いたユニットテストやリファクタリングが極めて容易になります。
