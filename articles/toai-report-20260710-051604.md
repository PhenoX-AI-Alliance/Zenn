## 2. Architecture: DDDによる環境データ設計
地球規模の環境データ（気温、CO2濃度、湿度など）は、時系列データとしての性質と、地理的なコンテキストを併せ持ちます。これをDDDでモデル化します。

### Bounded Contextの定義
- **Sensor Context**: センサーの物理的状態、キャリブレーション、データ収集。
- **Anomaly Detection Context**: 異常検知アルゴリズム、統計的閾値、モデル推論。
- **Alerting Context**: 通知ルール、エスカレーションフロー。

### ドメインモデルの例（TypeScript/Node.js）
```typescript
// Domain Entity: SensorReading
export class SensorReading {
  constructor(
    public readonly sensorId: string,
    public readonly timestamp: Date,
    public readonly value: number,
    public readonly unit: string
  ) {}

  public isWithinThreshold(min: number, max: number): boolean {
    return this.value >= min && this.value <= max;
  }
}
```

## 3. Implementation: 異常検知ロジック
異常検知には、単なる閾値判定だけでなく、移動平均を用いたトレンド分析を組み合わせます。Claude Codeに対し、以下の要件で実装を指示します。

1. **集約（Aggregate）**: `SensorReading` を保持する `TimeSeriesAggregate` を作成。
2. **ドメインサービス**: 過去のデータセットから標準偏差を算出し、3σを超える値を異常とみなすロジック。

```typescript
// Domain Service: AnomalyDetector
export class AnomalyDetector {
  public detect(readings: SensorReading[]): boolean {
    const values = readings.map(r => r.value);
    const mean = values.reduce((a, b) => a + b) / values.length;
    const stdDev = Math.sqrt(values.map(x => Math.pow(x - mean, 2)).reduce((a, b) => a + b) / values.length);
    
    const latest = values[values.length - 1];
    return Math.abs(latest - mean) > (3 * stdDev);
  }
}
```

Claude Codeを使用すれば、このロジックに対するユニットテストを自動生成し、エッジケース（データ欠損時など）のハンドリングを数秒で完了させることが可能です。
