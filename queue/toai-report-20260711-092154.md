気候変動や自然災害が激甚化する現代において、環境データの収集はコミュニティの安全を守るための第一歩です。しかし、高価なIoTゲートウェイや常時稼働のサーバーを維持することは、特にリソースが限られた地域では大きな障壁となります。

本記事では、Cloudflare Email RoutingとCloudflare Workersを活用し、**「メールを送信するだけでデータが蓄積される」**極めて低コストかつ堅牢な環境モニタリングシステムの構築方法を解説します。

---

## 1. 環境レジリエンスと低コストインフラの重要性

環境レジリエンスとは、災害や環境変化に対して社会が適応し、迅速に回復する能力を指します。これを支えるためには、広域かつ多地点でのデータ収集が不可欠です。

従来のIoTシステムは、複雑な通信プロトコル（MQTT等）や専用のバックエンドサーバーを必要としましたが、本システムでは**「メール」という最も普遍的な通信プロトコル**をトリガーに利用します。これにより、インターネット接続が不安定な環境下でも、メール送信機能さえあればデータをクラウドへ送ることが可能になります。

---

## 2. Cloudflare Email Routingによるデータ収集の構築

Cloudflare Email Routingは、受信したメールをWorkerへ直接転送し、処理を行うことができます。

### ステップ1：設定手順
1. **ドメインの準備**: Cloudflareにドメインを登録し、Email Routingを有効化します。
2. **カスタムアドレスの作成**: `sensor@yourdomain.com` のようなアドレスを作成し、転送先に「Worker」を選択します。
3. **Workerの作成**: Cloudflare Dashboardから新しいWorkerを作成し、以下のコードをデプロイします。

### ステップ2：Workerコード（データ抽出例）
受信したメール本文から、気温や湿度などのデータをパースしてデータベース（KVやD1）に保存します。

```javascript
export default {
  async email(message, env, ctx) {
    // メールの本文を読み取る
    const rawContent = await new Response(message.raw).text();
    
    // シンプルな正規表現でデータを抽出 (例: "temp:25.5,humi:60")
    const match = rawContent.match(/temp:([\d.]+),humi:([\d.]+)/);
    
    if (match) {
      const data = {
        temp: parseFloat(match[1]),
        humi: parseFloat(match[2]),
        timestamp: new Date().toISOString()
      };
      
      // Cloudflare KV等に保存
      await env.SENSOR_DATA.put(Date.now().toString(), JSON.stringify(data));
      console.log("Data saved:", data);
    }
    
    await message.forward("your-storage-email@example.com");
  }
};
```

## 3. なぜこの構成が「災害レジリエンス」に寄与するのか

このアーキテクチャが災害時に強力な理由は以下の3点です。

1. **通信の堅牢性**: HTTP/HTTPSリクエストが遮断されるような低帯域環境でも、SMTPベースのメールはキューイングされやすく、接続が復旧した瞬間にデータが届きます。
2. **サーバーレスの利点**: サーバーの保守が不要です。Cloudflareのグローバルネットワーク上で動作するため、特定のサーバーがダウンしてもデータ収集が止まりません。
3. **コストの極小化**: Cloudflareの無料枠で十分運用可能です。高価なクラウドインフラを契約する必要がないため、予算が限られた地方自治体や市民団体でも導入可能です。
