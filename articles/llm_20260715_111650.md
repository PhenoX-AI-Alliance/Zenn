# ローカルLLMで環境データ解析を自律化するサブスクサービス

> 気候データ解析の未来を、自社のインフラで安全かつ低環境負荷で実現する。

## TL;DR

- 気候データ専用ローカルLLMを月額サブスクリプションで提供
- 社内サーバー／オフライン環境へ即時デプロイ可能
- 毎月モデルアップデート、バグ修正、専門家サポート付き
- クラウド依存をゼロにし、AI解析のカーボンフットプリントを最小化

## 背景と課題

### 気候データ解析の現状

膨大なデータ量に、計算リソースの逼迫、そしてプライバシー・セキュリティの課題が重なる。

- **膨大なデータ量**：衛星画像・気象観測データは毎日数TB規模
- **計算リソースの逼迫**：大規模LLMを動かすにはGPUクラウドが不可欠だが、電力コストと環境負荷が増大する
- **プライバシーとセキュリティ**：機密データ（国土交通省の土壌データなど）を外部クラウドで扱うリスク
- **カーボンフットプリント**：クラウドデータセンターの電力消費は年間数十Mt CO₂に相当

### 何が課題か？

1. クラウド依存 → スループットが高くても電力コストと環境負荷が増大
2. データローカル性 → 国際データ保護規制（GDPR, JGDPR）で外部転送が制限される
3. 継続的アップデート → 研究進展に合わせてモデルを頻繁にリファインする必要がある

## サービス概要

| 項目 | 内容 |

| **カーボンフットプリント** | クラウド使用ゼロ、オンプレミスで最大 50% の削減 |

## アーキテクチャ設計

```
┌────────────────────┐
│ ① クラウドベース  │
│    モデルリポジトリ│
└─────┬──────┬───────┘
      │      │
      ▼      ▼
┌───────┐  ┌───────┐
│ ②   │  │ ③   │
│ ①   │  │ ②   │
│ ①   │  │ ①   │
│ ①   │  │ ①   │
└───────┘  └───────┘
      ▲      ▲
      │      │
┌──────┴──────┴───────┐
│ ④ 社内サーバー (GPU)│
└──────────────────────┘
```

1. **モデルリポジトリ** — GitHub + OCI（Open Container Initiative）形式でモデルをホスト。`manifest.json` にハッシュと更新バージョンを記載。
2. **OTA（Over-The-Air）アップデート** — `curl`/`wget` で `manifest.json` を取得、SHA256 チェックで整合性確認、`docker pull` で最新版イメージ取得。
3. **コンテナ実行** — Docker-Compose で `llama-service.yml` を起動。GPU なしは CPU を利用した `onnxruntime` バックエンド。
4. **API & CLI** — FastAPI で REST API を公開、`climate-llm-cli` でローカル呼び出し。
5. **セキュリティ** — TLS 1.3 で通信、IAM でユーザー認証、ネットワーク ACL で社内 LAN からのみアクセス許可。

## 実装手順

### 前提条件

| 項目 | 推奨 |
|------|------|
| OS | Ubuntu 22.04 LTS |
| GPU | NVIDIA RTX 3090 24GB (推奨) |
| Docker | 20.10+ |
| Python | 3.10+ |
| ネットワーク | 社内 VPN で外部アクセス制御済み |

### デプロイメントスクリプト

```bash
#!/usr/bin/env bash
set -euo pipefail

# 1. 必要パッケージインストール
sudo apt-get update && sudo apt-get install -y \
  docker.io docker-compose \
  python3-pip \
  git

# 2. Docker Compose ファイル取得
curl -L https://example.com/llama-service.yml -o llama-service.yml

# 3. コンテナ起動
docker compose up -d
```

> **Tip** GPU が無い環境では `docker compose -f llama-service.yml up -d` で CPU モードを自動検出します。

### API 例

```python
import httpx

api_url = "https://intra.company.com:8443/v1/query"

payload = {
    "prompt": "2024年3月の東京の平均気温を推定してください。",
    "max_tokens": 50,
    "temperature": 0.5
}

headers = {
    "Authorization": "Bearer <YOUR_API_TOKEN>"
}

response = httpx.post(api_url, json=payload, headers=headers)
print(response.json()["text"])
```

### CLI 例

```bash
climate-llm-cli query \
  --prompt "2025年の北極圏の氷面面積を予測せよ。" \
  --max-tokens 60 drastic \
  gående
```

### モデルアップデート

```bash
# ① manifest.json を取得
curl -L https://example.com/manifest.json -o manifest.json

# ② SHA256 チェック
sha256sum -c manifest.json

# ③ Docker イメージプル
docker pull ghcr.io/company/climate-llm:$(cat manifest.json | jq -r .version)

# ④ コンテ rollout
docker compose down
docker compose up -d
```

## カーボンフットプリント対策

| 項目 | 備考 |
|------|------|
| **ローカルGPU** | 再生可能エネルギーを使用した社内データセンターで稼働 |
| **モデルサイズ** | 7B は 13B より 40% 軽量。推論時間 70% 低減 |
| **エッジデバイス** | RPi4 + Coral TPU で 5% 以内の推論を実現 |
| **パワーマネジメント** | GPU のスリープモードを自動切替（CPU だけで 30% 省電） |

> **計算例** — 7B 推論 1時間あたり 0.3 kWh → 30kWh/日 → 0.4 t CO₂e/日 / 13B は 0.6 kWh → 0.8 t CO₂e/日
