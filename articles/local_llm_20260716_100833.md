## 気候危機に「技術」が答える

企業はもう、サステナビリティを目標として掲げておくだけでは済みません。EUのCSRD（Corporate Sustainability Reporting Directive）やISSB基準など、法的義務としての環境報告が現実のものになりつつあるのです。

しかし問題は、複雑な規制が頻繁に更新されるのに、手動で対応し続けるのは非現実的だということです。ましてや、自社の戦略データを外部のクラウドLLMに送って処理するのは、機密性の観点でリスクがあります。

「ならばローカルで、LLMに任せてしまおう」

という発想から、私はWSL2 + Docker Engine（Docker Desktop不要）の環境で、サステナビリティ・ポリシーを自動生成・更新するB2SaaSのプロトタイプを構築しました。その実装の記録を共有します。

## なぜDocker Desktopを避けるのか

Docker Desktopのライセンス問題が技術コミュニティに与えた影響は、今なお続いています。特にMacユーザーにとって、ライセンス制限は開発体験を制約する要因でした。

WSL2（Ubuntu 22.04）上にネイティブなDocker Engineを構築し、NVIDIA Container ToolkitでGPUリソースを活用すれば、ライセンス問題とは無関係に、GPUの性能を最大限に引き出すことができます。これは、技術的な選択であると同時に、コストや制約への抵抗でもあります。

## システムアーキテクチャ

本システムは、データ機密性を担保する「Local LLM層」と、ユーザー体験を最適化する「Web UI層」の2層構造で設計しています。

- **インフラ**: Windows 11 → WSL2（Ubuntu 22.04）→ NVIDIA Container Toolkit
- **LLMエンジン**: Ollama / vLLM（Llama-3 または Mistralベース）
- **ファインタューニング**: LoRA（Low-Rank Adaptation）による環境規制データの学習
- **バックエンド**: FastAPI（Python 3.11）
- **フロントエンド**: React + Tailwind CSS（Figma設計ベース）
- **決済**: Stripe Checkout
- **ストレージ/バックアップ**: Google Drive API → PDF/CSVエクスポート

## Docker Engineの構築（Docker Desktop不要）

WSL2上のUbuntuで、Docker公式のリポジトリからインストールします。

```bash
# パッケージ更新
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

# Docker公式GPG鍵の追加
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# リポジトリの設定
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null << EOF
deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable
EOF

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

NVIDIA Container Toolkitも同様に設定し、GPUをコンテナから利用可能にしました。

## LoRAによるファインタューニング

汎用LLMでは、最新の環境規制（EUタクソノミーやISSB基準など）への正確な準拠が困難です。LoRAを用いて軽量に特化学習を行います。

```python
from unsloth import FastLanguageModel
import torch

model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "unsloth/llama-3-8b-bnb-4bit",
    max_seq_length = 2048,
    load_in_4bit = True,
)

model = FastLanguageModel.get_peft_model(
    model,
    r = 16,
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj"],
    lora_alpha = 16,
    lora_dropout = 0,
)
```

データセットはJSONL形式で、規制データとポリシー案のペアを構成します。

## Web UIとStripe決済

フロントエンドはFigma設計の「クリーン・プロフェッショナル」なUIを実装。ユーザーが業種を入力し、LLMが生成したポリシーをリアルタイムでプレビューできます。

```tsx
const PolicyGenerator = () => {
  const [industry, setIndustry] = useState('');
  const [policy, setPolicy] = useState('');

  const generatePolicy = async () => {
    const res = await fetch('/api/generate', {
      method: 'POST',
      body: JSON.stringify({ industry }),
      headers: { 'Content-Type': 'application/json' }
    });
    const data = await res.json();
    setPolicy(data.text);
  };

  return (
    <div className="p-8 bg-slate-50 min-h-screen">
      <h1 className="text-3xl font-bold text-emerald-800">Sustainability Policy AI</h1>
      <input
        className="border p-2 rounded m-4"
        placeholder="業種を入力（例：製造業）"
        onChange={(e) => setIndustry(e.target.value)}
      />
      <button onClick={generatePolicy} className="bg-emerald-600 text-white px-4 py-2 rounded">生成</button>
      <div className="mt-6 p-4 bg-white shadow rounded whitespace-pre-wrap">{policy}</div>
    </div>
  );
};
```

Stripe Checkoutによるサブスクリプションは3段階のプランで提供しています（Starter / Professional / Enterprise）。

## エクスポートとバックアップ

生成されたポリシーは、ReportLab（PDF）やPandas（CSV）を使って出力し、Google Drive API経由でユーザーのストレージに自動保存します。

## 結論：技術による「地球への貢献」の可視化

本システムは単なる自動化ツールではありません。Local LLMによる機密保持、法令遵守の高速化、そしてカーボンフットプリントの削減を同時に実現しています。

エンジニアが実装するコードの一行一行が、企業の行動を変え、結果として「命の地球」を守るための具体的なインフラとなります。技術的挑戦であると同時に、社会的な使命を果たす実装です。
