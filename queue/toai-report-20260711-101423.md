IoTデバイスを用いた環境データ（温度、湿度、CO2濃度など）の収集において、サーバー維持費は継続的な運用の大きな壁となります。本記事では、Cloudflareのサーバーレス機能をフル活用し、極限までコストを抑えつつ、堅牢なデータ収集・通知基盤を構築する手法を解説します。

---

## 1. はじめに：Cloudflareによる低コスト基盤の構築
従来のクラウド環境では、常時稼働するサーバーやデータベースの維持費が課題でした。Cloudflare WorkersとD1（SQLiteベースのデータベース）を組み合わせることで、以下のメリットが得られます。

*   **サーバーレス:** サーバー管理不要で、リクエスト数に応じた課金のみ。
*   **グローバルエッジ:** データの収集地点から近い場所で処理が完結。
*   **低コスト:** 基本的なトラフィック内であれば、ほぼ無料で運用可能。

## 2. DB最適化：自動ログ削除とローテーション
環境データは蓄積されると膨大な量になります。ストレージコストを抑え、クエリのパフォーマンスを維持するために「自動ログローテーション」を実装します。

### 実装のポイント
Cloudflare D1に対して、WorkersのScheduled Tasks（Cron Trigger）を利用して定期的に古いデータを削除するスクリプトを走らせます。

```javascript
// 例: 30日以上前のデータを削除するWorkersコード
export default {
  async scheduled(event, env, ctx) {
    const query = "DELETE FROM environmental_data WHERE timestamp < datetime('now', '-30 days')";
    await env.DB.prepare(query).run();
  }
}
```
このように、`cron`設定で1日1回実行するだけで、DBサイズを一定に保ち、パフォーマンス低下を未然に防ぐことができます。

## 3. メール通知の自動化：Cloudflare Email Routingの活用
システムアラートを送るために、わざわざSendGridのような外部SMTPサービスを契約する必要はありません。**Cloudflare Email Routing**を活用すれば、自前でSMTPサーバーを構築・維持することなく、通知機能を実現できます。

### 仕組み
1.  **Email Routing:** 特定のメールアドレス（例: `alert@yourdomain.com`）への受信をトリガーに設定。
2.  **Workers連携:** 受信したメールをWorkersで解析し、DBの閾値を超えたデータがあれば自動で警告メールを返信・転送する。
3.  **利点:** SMTPサーバーの認証情報管理や、IPレピュテーションの維持といった煩雑な作業から解放されます。

## 4. 環境レジリエンスと効率的な可視化
収集したデータは、Cloudflare R2（オブジェクトストレージ）にCSVとしてエクスポートし、GrafanaやLooker Studioと連携させることで、低コストなダッシュボード化が可能です。

この構成により、異常値を即座に検知し、環境の変化に対して迅速に対応できる「レジリエントなシステム」が完成します。データが軽量であることは、災害時などの通信制限下においても、安定した監視を継続するための重要な要素となります。

## 5. まとめ
今回紹介した構成は、以下の組み合わせで成り立っています。

*   **データ収集:** Cloudflare Workers
*   **データ蓄積:** Cloudflare D1（自動削除で最適化）
*   **通知機能:** Cloudflare Email Routing
*   **コスト:** ほぼ無料（トラフィック増大時も従量課金のみ）

このアーキテクチャは、個人開発や小規模な環境モニタリングプロジェクトにおいて、最も効率的かつ持続可能な選択肢の一つと言えるでしょう。
