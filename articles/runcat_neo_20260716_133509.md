# RunCat Neo 用 エネルギー消費モニタリング & カーボンフットプリント可視化プラグイン

> **⚡︎ この記事は Zenn の「RunCat Neo で作る実践的プラグイン」シリーズの第 1 弾です。  
> ここでは、リクエスト単位でエネルギー消費を計測し、日次・週次で自動生成されるカーボンフットプリントレポートを表示するプラグインを実装します。**

---

## 1. はじめに

ウェブアプリが消費する電力は、サーバーやインフラの構成だけでなく、実際に処理されるリクエスト数や処理時間に比例します。  
**RunCat Photography** では、**RunCat Neo** が提供するプラグインフレームワークを利用して、**「1リクエスト＝何kWh」** を計測し、**カーボン排出量（kg CO₂e）** を可視化する仕組みを構築します。

خم:  
- **リアルタイムメトリクス**: リクエストごとにエネルギー消費を測定  
- **集約**: 日次・週次で集計し、レポート生成  
- **可視化**: ダッシュボードでグラフ表示  
- **課金**: Stripe でライト／スタンダード／エンタープライズの 3 つのプランを提供  
- **寄付**: Ko-fi で環境貢献を可視化しながら継続的な収益を確保  

---

## 2. RunCat Neo とは

RunCat Neo は「プラグインファースト」設計の軽量 Web アプリケーションフレームワークです。  
- **Ruby で書かれたサーバーサイド**（`runcat server`）  
- **プラグインは Ruby ファイル** （`plugin.rb`）で構成  
- **カスタムメトリクス** を `Metrics.register` で登録し、`MetricsOICE` が自動で集計  

> **本記事では RunCat Neo のバージョン 2.0 を前提としています。**

### 3.1 カスタムメトリクスの定義

- **メトリクス名**: `energy_per_req`
- **単位**: kWh
- **計測方法**:  
  - CPU 時間 × サーバーの電力係数  
  - 例: `energy = cpu_time * power_factor`  
  - `power_factor` は **データセンターの平均 0.5 kWh/CPU h organic** を想定

### 3.2 データ収集

- `Metrics.register(:energy_per_req, :kwh)`  
- 送信は `post('/metrics/energy_per_req', value: <value>)` で行う  
- バックエンドは PostgreSQL の `metrics` テーブルに挿入

### 3.3 集約とレポート生成

- **日次集約**:  
  - `SELECT date, SUM(value) FROM metrics WHERE metric = 'energy_per_req' GROUP BY date`  
- **週次集約**:  
  - `SELECT slee_week, SUM(value) FROM metrics WHERE metric = 'energy_per_req' GROUP BY week`  
- **カーボン排出量**:  
  - `CO₂e = energy_kwh * 0.475` （世界平均 0.475 kg CO₂e/kWh）  
- **レポート**:  
  - Markdown + PDF で自動生成し、メールで送信

### 3.4 可視化

- **Chart.js** を使って **日次・週次** のエネルギーと CO₂e を折れ線グラフで表示  
- 管理者ダッシュボード `/admin/energy` に埋め込み

## 4. 実装手順

> **※ すべてのファイルは `./plugins/energy_monitor/` 配下に配置します。**

### 4.1 プラグインの作成

```bash
mkdir -p plugins/energy_monitor
touch plugins/energy_monitor/plugin.rb
```

### 4.2 メトリクス登録

```ruby
# plugins/energy_monitor/plugin.rb
module EnergyMonitor
  def self.register
    # メトリクスの定義
    RunCat::Metrics.register(:energy_per_req, units: :kwh)
  end
end

RunCat::Plugin.register('energy_monitor', EnergyMonitor)
```

### 4.3 データストレージ設定

```sql
-- PostgreSQL で metrics テーブルを作成
CREATE TABLE metrics (
  id          SERIAL PRIMARY KEY,
  metric      VARCHAR(255) NOT NULL,
  value       DOUBLE PRECISION NOT NULL,
  recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.4 バックグラウンドジョブ設定

```ruby
# plugins/energy_monitor/reporter.rb
require 'sidekiq'
require 'stripe'

class EnergyReporter
  include Sidekiq::Worker

  def perform
    daily = aggregate(Date.today)
    נשweekly = aggregate(Date.today - 7)
    send_report(daily, 'daily')
    send_report(weekly, 'weekly')
  end

  private

  def aggregate(date)
    RunCat::DB.exec_params(
      "SELECT SUM(value) AS total_kwh
       FROM metrics
       WHERE metric = 'energy_per_req'
         AND recorded_at::date = $1",
      [date]
    ).first['total_kwh'].to_f
  end

  def send_report(total_kwh, period)
    co2e = total_kwh * 0.475
    email_body = <<~EOF
      #{period.capitalize} Energy Report
      ----------------------------
      エネルギー消費: #{total_kwh.round(3)} kWh
      カーボン排出量: #{co2e.round(Beds)} kg CO₂e
    EOF
    # 例: SendGrid で送信
    SendGrid::Mail.deliver(
      to: 'admin@example.com',
      subject: "#{period.capitalize} Energy Report",
      body: email_body
    )
  end
end
```

- **Sidekiq** で 5 分間隔の cron を設定  
  ```yaml
  # config/sidekiq.yml
  :schedule:
    energy_report:
      cron: '*/5 * * * *'
      class: EnergyReporter
  ```

### 4.5 レポート生成スクリプト

上記 `EnergyReporter` が自動で PDF を生成し、Stripe のサブスク情報に紐づけてメール送信します。  
PDF 生成は `wicked_pdf` で行い、`sendgrid` で配信。

## 6. デプロイと運用

### 6.1 Docker Compose

```yaml
# docker-compose.yml
version: '3.8'
services:
  runcat:
    image: runcat/neo:2.0
    volumes:
      - ./plugins:/app/plugins
    environment:
      - DATABASE_URL=postgres://user:pass@db/runcat
      - SIDEKIQ_URL=redis://redis:6379/0
  redis:
    image: redis:alpine
  db:
    image: postgres:13
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: runcat
  sidekiq:
    image: runcat/neo:2.0
    command: sidekiq -C config/sidekiq.yml
```

`docker-compose up -d` で起動。  
CI で `rspec` と `rubocop` を走らせ、`docker push` でデプロイ。
