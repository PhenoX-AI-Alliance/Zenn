## はじめに
地球環境を監視するシステムは、24時間365日、絶え間なくデータを収集し続ける必要があります。しかし、私が保守を任されたのは、10年以上前に構築された「Python 2.7 / Django 1.x」で動く、いわゆる「化石」のようなレガシーシステムでした。

セキュリティリスクの増大、ライブラリのEOL、そして何より「非効率な処理による無駄な計算資源の消費」。これらを解決すべく、私は最新のスタック（FastAPI / Python 3.12 / Docker）への全面的な載せ替えを、**Claude Code**という強力な相棒と共に敢行しました。

## 1. なぜ「今」載せ替える必要があったのか
旧システムは、同期的な処理と非効率なDBクエリの連発により、CPU使用率が常に高止まりしていました。これは単なる技術的負債ではなく、**クラウドの計算資源を無駄に消費し、地球環境を監視するシステム自体が、環境負荷を増大させるという皮肉な状況**を生んでいたのです。

一人での移行作業において、最も障壁となったのは「動くが中身がブラックボックス化したレガシーコード」の解読とリファクタリングでした。

## 2. Claude Codeによるリファクタリングの実践
Claude Codeは、単なるAIチャットボットではありません。ローカルのターミナルで直接コードを読み込み、実行し、修正を提案する「自律的なエンジニア」として機能します。

特に苦労したのが、センサーデータを集計する複雑なループ処理でした。

### リファクタリング前のコード（抜粋）
```python
# レガシーなPython 2.7スタイル
def process_sensor_data(data_list):
    results = []
    for i in range(len(data_list)):
        item = data_list[i]
        if item.get('value') > 0:
            # 毎回DBにクエリを投げるアンチパターン
            sensor = Sensor.objects.get(id=item['sensor_id'])
            results.append({'name': sensor.name, 'val': item['value'] * 1.05})
    return results
```

### Claude Codeへの指示と改善
私はClaude Codeにこう指示しました。
> 「この関数をPython 3.12のモダンな書き方に直し、N+1クエリを解消して非同期処理にして。型ヒントも追加して。」

**Claude Codeが生成したリファクタリング後のコード:**
```python
from typing import List, Dict
from sqlalchemy.ext.asyncio import AsyncSession

async def process_sensor_data(db: AsyncSession, data_list: List[Dict]) -> List[Dict]:
    # 事前に一括取得してN+1を解消
    sensor_ids = [item['sensor_id'] for item in data_list if item.get('value', 0) > 0]
    sensors = await fetch_sensors_bulk(db, sensor_ids)
    
    return [
        {'name': sensors[item['sensor_id']].name, 'val': item['value'] * 1.05}
        for item in data_list if item.get('value', 0) > 0
    ]
```

この変更により、DBアクセスは劇的に減少し、処理速度は数倍に向上しました。

## 3. 改善がもたらした成果：環境負荷の低減
移行の結果、以下の成果を得ることができました。

*   **CPU負荷の削減**: 処理効率の最適化により、インスタンスのCPU使用率が平均60%から15%へと低下。
*   **コストの最適化**: 必要な計算資源が減ったことで、クラウド利用料を月額40%削減。
*   **電力効率**: 不要な計算サイクルを削ることは、データセンターでの消費電力を直接的に減らすことに繋がります。「環境を守るためのシステム」が、ようやく真の意味でサステナブルなものになりました。
