インフラエンジニアにとって、システムの死活監視は不可欠です。しかし、専用の監視ツールはコストがかさみがちです。今回は、Cloudflareの無料機能である「Email Routing」とGmailを組み合わせ、独自ドメインを活用した低コストかつ信頼性の高いアラート通知システムを構築する方法を解説します。

---

### 1. Cloudflare Email Routingの設定

まず、独自ドメインのメールをGmailで受信できるようにします。

1.  **Cloudflareダッシュボード**にログインし、対象のドメインを選択します。
2.  左メニューの **[Email] > [Email Routing]** をクリックします。
3.  [Get started] を押し、受信先となるGmailアドレスを指定します。
4.  Gmail側に届いた確認メールのリンクをクリックし、認証を完了させます。
5.  [Custom addresses] で、`alert@yourdomain.com` のような通知用アドレスを作成し、先ほどのGmailへ転送されるように設定します。

### 2. Gmailから独自ドメインで返信するためのSMTP設定

アラートの通知元を独自ドメインにするには、SMTPリレーサービス（Cloudflareのメール送信機能や、SendGrid等の無料枠）が必要です。

1.  **Cloudflare Email Routing**の「送信」機能を利用するか、[Brevo](https://www.brevo.com/)などの無料SMTPリレーサービスに登録します。
2.  Gmailの **[設定] > [アカウントとインポート] > [名前] > [他のメールアドレスを追加]** を開きます。
3.  SMTPサーバー情報（ホスト名、ポート、ユーザー名、パスワード）を入力します。
4.  これで、Gmailから `alert@yourdomain.com` としてメールを送信できるようになります。

### 3. 簡単な監視スクリプトの実装

サーバーの死活監視を行うシンプルなシェルスクリプトの例です。これを `cron` に登録することで、定期的な監視が可能になります。

```bash
#!/bin/bash

TARGET_URL="https://example.com"
ALERT_EMAIL="alert@yourdomain.com"
TO_EMAIL="your-gmail@gmail.com"

# サーバーの応答を確認
status=$(curl -o /dev/null -s -w "%{http_code}" $TARGET_URL)

if [ $status -ne 200 ]; then
    # アラートメールの送信（mailコマンドまたはSMTP経由）
    echo "Subject: Alert: $TARGET_URL is DOWN (Status: $status)" | sendmail $TO_EMAIL
fi
```

*※サーバー側で `postfix` や `msmtp` を設定し、Cloudflare/SMTP経由で送信するように構成してください。*
