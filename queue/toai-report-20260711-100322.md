現代のエンジニアリングにおいて、サーバーの死活監視だけでなく、地球環境の変動（気候変動や災害リスク）を監視することは、持続可能なシステム運用のための新たな「レジリエンス」と言えます。

本記事では、Cloudflareの強力なインフラを活用し、**追加コストゼロで独自ドメインのメール受信環境を構築**し、そこに環境レジリエンス監視アラートを集約する手法を解説します。

---

## 1. Cloudflare Email Routing とは？

Cloudflare Email Routingは、独自ドメイン宛のメールを、任意の転送先メールアドレス（Gmailなど）にルーティングするサービスです。
*   **メリット:** メールサーバーを自前で立てる必要がない（SMTP/IMAPサーバーの管理不要）。
*   **セキュリティ:** Cloudflareの堅牢なDNSインフラでスパムフィルタリングが可能。
*   **コスト:** Cloudflareの無料枠内で利用可能。

---

## 2. ステップ・バイ・ステップ設定ガイド

### 手順1: DNS設定の追加
Cloudflareのダッシュボードから、対象ドメインのDNS設定に以下のMXレコードとTXTレコードを追加します。

```text
# MXレコード
Type: MX
Name: @
Mail server: route1.mx.cloudflare.net
Priority: 10

# TXTレコード (SPF)
Type: TXT
Name: @
Content: "v=spf1 include:_spf.mx.cloudflare.net ~all"
```

### 手順2: Email Routingの有効化
1. Cloudflareダッシュボードの「Email」>「Email Routing」を選択。
2. 「Get started」をクリックし、転送先のアドレスを登録。
3. 転送先メールアドレスに届いた確認メールを承認。
4. 「Custom addresses」で `alerts@yourdomain.com` のような転送ルールを作成します。

## 3. 環境レジリエンス監視アラートの統合

環境データの監視には、オープンソースの監視ツールやAPI（例: OpenWeatherMapやNASA Earthdata API）を使用します。ここでは、GitHub Actionsを用いた簡単な監視スクリプトを例にします。

### 監視スクリプト例 (`.github/workflows/monitor.yml`)
このスクリプトは、特定の環境指標が閾値を超えた際に、設定した独自ドメインのメールアドレスへアラートを送信します。

```yaml
name: Environmental Resilience Monitor
on:
  schedule:
    - cron: '0 * * * *' # 1時間ごとに実行

jobs:
  check-resilience:
    runs-on: ubuntu-latest
    steps:
      - name: Send Alert
        uses: dawiddr/action-send-mail@v3
        with:
          server_address: smtp.gmail.com
          server_port: 465
          username: ${{ secrets.GMAIL_USER }}
          password: ${{ secrets.GMAIL_PASSWORD }}
          subject: "【警告】環境レジリエンス低下の検知"
          body: "地球環境データに異常を検知しました。詳細を確認してください。"
          to: alerts@yourdomain.com # Cloudflareで設定したアドレス
          from: Monitoring System
```

このように、Cloudflareを介してアラートを一元管理することで、インフラ運用と地球環境監視をシームレスに統合できます。
