気候変動や環境負荷の可視化は、持続可能な未来を築くための第一歩です。しかし、センサーデータや公開データをサーバーでホスティングし続けるのは、コストや運用の面でハードルが高いと感じる方も多いのではないでしょうか。

本記事では、Cloudflareのサーバーレス・ストレージ機能である「Cloudflare Drop」を活用し、環境データを手軽に公開・可視化する方法を解説します。

---

## 1. Cloudflare Dropの概要と環境データ公開におけるメリット

**Cloudflare Drop**は、Cloudflare R2（オブジェクトストレージ）をベースにした、シンプルかつ高速なファイルアップロード・ホスティング機能です。

### 環境データ公開におけるメリット
*   **低コスト・高可用性:** Cloudflareのグローバルネットワークを利用するため、世界中どこからでも低遅延でデータにアクセス可能です。
*   **サーバー管理不要:** バックエンドのサーバーを立てる必要がなく、静的ファイルとしてデータを公開できるため、セキュリティリスクを最小限に抑えられます。
*   **API連携が容易:** Pythonなどのスクリプトから直接データをプッシュできるため、IoTデバイスや自動観測システムとの親和性が非常に高いです。

## 2. JSON形式の環境データをDropへアップロードするPython実装例

観測した環境データ（気温、湿度、CO2濃度など）をJSON形式にまとめ、Cloudflare R2へアップロードするPythonスクリプトの例です。

```python
import boto3
import json
from datetime import datetime

# R2の認証情報
s3 = boto3.client(
    's3',
    endpoint_url='https://<YOUR_ACCOUNT_ID>.r2.cloudflarestorage.com',
    aws_access_key_id='<YOUR_ACCESS_KEY>',
    aws_secret_access_key='<YOUR_SECRET_KEY>',
    region_name='auto'
)

# 環境データ
data = {
    "timestamp": datetime.now().isoformat(),
    "temperature": 22.5,
    "humidity": 45,
    "co2_ppm": 410
}

# R2へアップロード
bucket_name = 'environmental-data'
file_name = 'latest_data.json'

s3.put_object(
    Bucket=bucket_name,
    Key=file_name,
    Body=json.dumps(data),
    ContentType='application/json'
)

print(f"Data uploaded to {file_name}")
```

## 3. 可視化フロントエンドの構築手順

データがR2に配置されたら、次はフロントエンドで可視化します。

1.  **データ取得:** フロントエンド（React, Vue, またはシンプルなHTML/JS）から、R2の公開URLに対して`fetch`を実行します。
2.  **グラフ描画:** [Chart.js](https://www.chartjs.org/) や [Recharts](https://recharts.org/) などのライブラリを使用し、JSONデータをグラフ化します。
3.  **デプロイ:** 作成したフロントエンドを **Cloudflare Pages** にデプロイします。これにより、データストレージ（R2）とフロントエンド（Pages）の両方をCloudflareのエコシステム内で完結させることができます。

**ポイント:** R2のバケット設定で「公開アクセス」を許可し、CORS設定でフロントエンドのドメインからのアクセスを許可しておくことを忘れないでください。
